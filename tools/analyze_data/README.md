# Extracted data analyzer

Analyzes the files generated by `tools/extract_data/` and creates an inventory by:

- extension;
- magic signature;
- known format;
- category;
- size;
- possible Nitro compression;
- approximate entropy;
- priority for the first resource viewer.

It does not modify the extracted files.

## Usage

First locate the folder created by the extractor:

```text
data/extracted/<first 16 characters of the SHA-256>/
```

From the repository root:

```powershell
py .\tools\analyze_data\analyze_data.py ".\data\extracted\<ID>"
```

You can also specify a different output folder:

```powershell
py .\tools\analyze_data\analyze_data.py ".\data\extracted\<ID>" `
  --output ".\data\reports\<ID>"
```

## Output

```text
analysis/
├── asset_inventory.csv
├── summary.json
└── report.md
```

### `asset_inventory.csv`

One row per file:

- `file_id`
- `path`
- `extension`
- `size`
- `sha256`
- first bytes in ASCII and hexadecimal
- detected format
- category
- compression indicator
- entropy
- score and reason

You can open it directly with Excel or LibreOffice.

### `summary.json`

Summary that other tools can process:

- count by format;
- count by category;
- count by extension;
- compressed files;
- main candidates.

### `report.md`

Readable report with the recommended candidates for the first viewer.

## Initially recognized signatures

### Nitro 3D

- `BMD0` — NSBMD, model
- `BTX0` — NSBTX, textures
- `BCA0` — NSBCA, skeletal animation
- `BTA0`, `BTP0`, `BMA0`, `BVA0` — auxiliary animations

### Nitro 2D

- `RGCN` — NCGR, graphics
- `RLCN` — NCLR, palette
- `RCSN` — NSCR, tilemap
- `RECN` — NCER, sprites/cells
- `RNAN` — NANR, animation

### Others

- `NARC`
- `SDAT`, `SSEQ`, `SSAR`, `SBNK`, `SWAR`, `SWAV`
- PNG, JPEG, BMP, RIFF, Ogg, ZIP and Gzip
- Nitro LZ77, Huffman and RLE headers

## Which candidate to choose

The recommended first target is:

1. a small `NSBTX`, if one appears;
2. an `NCGR` + `NCLR` pair;
3. an `NSBMD` once textures are available;
4. a `NARC` only when we need to open container files.

The tool prioritizes recognized graphical resources over large unknown files.
